# 프로젝트 개요

## 1. 목표

`green-house`는 비닐하우스 난 농장의 운영 기록을 웹에서 관리하기 위한 시스템이다.

주요 목적은 다음과 같다.

- 농장 위치, 난 묶음, 작업 이력, 판매 전표를 디지털화한다.
- 현장 작업자가 이해하기 쉬운 농장맵과 큰 버튼 중심 UI를 제공한다.
- 관리자에게 농장 현황, 작업 기록, 판매/경매/입금 상태를 한 곳에서 보여준다.
- 종이 전표와 A5 출력 흐름을 유지하면서 데이터를 누적한다.

이 프로젝트의 목표는 완전 자동화가 아니라 **현장에서 꾸준히 쓸 수 있는 관리 시스템**이다.

## 2. 사용자

### 현장 작업자

- 농장 위치와 난 묶음을 직관적으로 확인한다.
- 마우스/터치 기반으로 선택, 이동, 작업 기록을 남긴다.
- 복잡한 입력보다 큰 버튼, 확인창, 출력물을 선호한다.

### 관리자

- PC/태블릿에서 데이터를 입력하고 정리한다.
- 난 묶음 위치, 작업 이력, 판매 전표, 경매 정산을 관리한다.
- 기존 수기 기록과 전표를 시스템에 옮기고 백업을 관리한다.

## 3. 현재 관리 범위

```text
농장 구조
→ 동 / 물리 배드 / 논리 구역 / 세그먼트 / 난 묶음

작업 관리
→ 작업 유형 / 작업 이력 / 위치 이동 이력

판매 관리
→ 거래처 / 판매 전표 / 판매 품목 / A5 출력

경매 관리
→ 경매 lot / 경매 결과 / 반환 확인 / 경매 정산

입금 관리
→ 전표 입금 / 정산 입금 / 거래처 잔액 / 입금 이벤트
```

## 4. 운영 흐름

초기 운영은 다음 흐름을 기준으로 한다.

```text
현장 작업 또는 수기 전표 발생
→ 관리자 또는 작업자가 시스템에 입력
→ 농장 현황/작업 이력/판매 관리에서 조회
→ 필요 시 A5 출력
→ 데이터 백업
```

농장 위치 관리는 다음 흐름을 기준으로 한다.

```text
농장 현황에서 전체 상태 확인
→ 동/배드/구역 단위로 조회
→ 수정이 필요하면 난 묶음 관리 화면으로 이동
→ 난 묶음 생성/수정/이동
→ 위치 이동 이력 자동 기록
```

판매·경매 관리는 다음 흐름을 기준으로 한다.

```text
일반 거래처 판매
→ 판매 전표 생성
→ 입금 확인
→ 거래처 잔액 반영

경매장 출하
→ 경매 lot 생성
→ 경매 결과/유찰/반환 추적
→ 경매 정산 생성
→ 입금 확인
```
