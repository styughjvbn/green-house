# 기능 요약

## 1. 대시보드

농장 운영 상태를 빠르게 확인하는 진입 화면이다.

표시 대상:

- 전체 동/배드/구역 수
- 전체 난 묶음 수
- 상태 이상 수
- 분갈이 예정 수
- 최근 작업
- 판매/입금 요약
- 빠른 이동 링크

대시보드는 상세 수정 화면이 아니다.

## 2. 농장 현황

전체 농장을 조회하는 화면이다.

핵심 역할:

```text
전체 농장의 난 묶음 위치·빈 구간·상태 분포 확인
→ 정규화 배치에서 동별 점유 현황 비교
→ 실제 배치에서 현장 위치 관계 확인
→ 동·물리 다이·논리 구역·난 묶음 선택
→ 오버레이 패널에서 선택 범위와 상세 확인
```

기본 보기는 모든 동을 같은 크기로 비교하는 정규화 배치다. 실제 크기가
아닌 비교용 배치임을 화면에 표시하며, 실제 배치는 현장 위치 관계 확인을
위한 보조 보기로 제공한다.

난 묶음 위치 블록은 전체 농장 수준부터 표시한다. 확대 단계가
`농장 / 동 / 다이 / 구역`으로 바뀔 때 라벨 정보량만 늘리며, 낮은 단계에서는
상태·선택·검색 결과·빈 구간을 우선 표시한다. 채움색 기준은 상태·품종·년생
중 선택하고, 상태 테두리와 선택·검색 강조는 별도로 유지한다.

검색·보기 설정·범례·줌은 지도 위 오버레이로 제공한다. 상세 패널도 지도
오버레이로 열며 선택 위치가 패널 뒤에 가려지지 않도록 지도 중심을 보정한다.

농장 현황은 조회와 위치 파악에 집중한다.

하지 않는 작업:

- 난 묶음 생성
- 난 묶음 수정
- 난 묶음 삭제
- 드래그 이동

수정이 필요하면 난 묶음 관리 화면으로 이동한다.

## 3. 난 묶음 관리

난 묶음을 실제로 생성, 수정, 삭제, 이동하는 작업 화면이다.

주요 기능:

- 전체 농장의 물리 다이를 동 번호와 다이 표시 순서 기준으로 연속 탐색
- 이전·다음 이동, 동의 첫 다이 빠른 이동
- 한 화면에 물리 다이 2~4개 표시 및 다이 단위 스와이프
- URL의 `startBedId`, `bedCount`로 현재 위치와 표시 개수 복원
- 배드별 좌/우 논리 구역 표시
- 난 묶음 등록/수정/삭제
- 선택한 난 묶음 정보 기반 복사 등록
- 품종별 난 묶음 색상 구분 토글
- 현재 표시 다이 범위의 난 묶음 목록과 작업 요약
- 검색어가 없을 때 현재 표시 범위 목록 유지
- 검색어 입력 시 현재 목록 또는 전체 농장 범위를 선택해 결과만 조회
- 검색어와 관련된 자동 그룹·사용자 그룹을 함께 제안하고 선택한 그룹의 구성원 목록 조회
- 화면·전체·그룹 검색 범위는 배타적으로 선택하며 그룹 선택 시 해당 구성원만 맵에서 활성화
- 다이 이동 후에도 선택 난 묶음 유지 및 화면 밖 상태 안내
- 난 묶음 등록/수정은 품종 마스터에서 검색 선택한 품종 기준으로만 처리한다.
- 다른 동 이동 목적지 선택
- 화분 크기는 표준 규격 선택값만 신규 저장
- 이동 후 위치 이동 이력 자동 생성
- 배드 정밀 설정/자리 차지 정보 확인
- 선택 난 묶음의 사용자 그룹 생성·추가·해제
- 사용자 그룹별 묶음 수와 총 수량 확인, 사용 종료 그룹 보관
- 현재 표시 범위의 난 묶음을 품종·현재 년생·화분 크기로 자동 그룹화
- 자동 그룹의 묶음 수·총수량·위치 수 확인과 구성원 일괄 선택
- 난 묶음 목록 체크박스 다중 선택 후 사용자 그룹 일괄 추가·해제

## 4. 작업 이력

농약, 비료, 분갈이, 정리, 상태 기록, 메모, 위치 이동을 기록한다.

주요 기능:

- 작업 이력 등록
- 대상 유형 선택
- 기간/대상/작업 유형 필터
- 작업 유형 추가/수정/비활성화/정렬
- 위치 이동 이력 자동 생성
- 입고 기록처럼 작업별 필드가 다른 이력은 상세 정보를 함께 표시
- 실수로 생성한 작업 이력 취소 처리
- 취소된 작업 이력 기본 목록 숨김

운영 원칙:

- 작업 이력은 가능하면 삭제하지 않는다.
- 잘못 입력한 경우 취소 상태와 취소 사유를 남기는 방향을 우선한다.
- 동 전체 농약 작업은 저장 시 실제 난 묶음 대상과 위치 스냅샷을 생성한다.
- 농약 작업 대상은 동 전체·자동 그룹·사용자 그룹·직접 다중 선택에서 고를 수 있다.
- 그룹 선택 후 실제 대상 미리보기에서 개별 난 묶음을 제외하고 저장할 수 있다.
- 여러 날 이어지는 작업은 하나의 작업으로 시작·일시중지·재개한다.
- 대상 난 묶음별 시작·완료·건너뛰기와 전체 진행률을 관리한다.
- 완료된 상위 작업은 각 대상 난 묶음 이력에서 전파 출처와 함께 조회한다.
- 난 묶음 이동 후에도 작업 당시 위치와 현재 위치를 구분해 표시한다.
- 다중 생성 작업으로 여러 위치의 난 묶음을 한 번에 생성하고 사용자 그룹 소속과 원인 작업을 함께 남긴다.
- 다중 생성 취소 전 다른 작업·이동·판매·입고 연결을 검사하고, 연결이 없으면 생성 결과와 사용자 그룹 소속을 안전하게 취소한다.
- 난 묶음 상세에서 직접 원본·결과 계보와 관계를 만든 작업·당시 수량을 조회한다.
- 다중 생성 등 작업 효과로 생성된 난 묶음에서도 원인 작업을 통합 이력으로 조회한다.
- 부분·전체 분갈이에서 투입 수량과 손실을 검증하고 여러 결과 난 묶음, 선택한 사용자 그룹 상속, 원본·결과 계보를 함께 기록한다.
- 난 묶음 목록에서 원본을 선택해 분갈이 화면을 열고, 결과별 위치·수량·화분·년생과 손실 사유를 입력하며 수량 일치와 원본 잔여량을 즉시 확인한다.
- 완료된 구조 변경 작업에 별도 보정 작업과 사유를 연결하고, 원본 결과를 삭제하지 않은 채 어떤 작업이 보정했는지 조회한다.
- 원본 구조 변경 작업이 만든 결과에 한해 수량·상태를 보정하고 전후 값을 남기며, 후속 작업·이동·판매·입고 연결이 있는 결과는 변경을 차단한다.
- 월간 기간 작업 캘린더에서 상태를 필터링하고 저장된 작업에 다시 진입해 전체·대상별 진행 상태를 관리한다.
- 기간 작업의 실행 회차·대상 완료·전체 완료 시 실제 완료일을 입력한다. 기본값은 농장 기준 오늘이며 과거 날짜로 수정할 수 있다.
- 기존 작업 기록 입력 화면과 난 묶음 빠른 작업은 신규 작업 실행 API로 즉시 완료 기록을 생성한다.
- 난 묶음 통합 이력에서 구조 변경 결과의 보정 화면을 열어 수량·상태와 사유를 입력하고 전후 이력을 확인한다.
- 작업 관리 페이지는 `WorkOperation` 조회·등록을 제공하는 `작업 목록`, 월간 `캘린더`, 완료·취소·보정 작업을 조회하는 `작업 이력` 탭으로 구분한다. 작업 등록은 작업 유형을 먼저 고른 뒤 작업 기록 또는 작업 계획 방식을 선택하고, 결과 입력이 필요한 구조 변경은 계획으로만 등록한다.
- 기간 작업은 일반 기록형 작업과 자리 이동·분갈이·분주·합식·입고 포트 작업을 함께 계획한다. 구조 변경은 그룹 대상을 실제 난 묶음으로 확정하고 여러 실행 회차에서 일부 수량과 복수 결과를 관리한다.
- 구조 변경 실행 입력은 계획 잔여 수량, 원본 속성, 현재 배치를 기본값으로 추론하며 사용자는 제외 대상·부분 수량·결과 분리·분주 후보 같은 예외를 수정한다.

## 5. 판매 관리

거래처와 판매 전표를 관리한다.

주요 기능:

- 거래처 등록/조회
- 판매 전표 생성
- 판매 전표 취소
- 판매 품목 입력
- 판매 품목별 난 묶음 다중 배분
- 품목 금액과 전표 총액 자동 계산
- 판매 전표 목록/상세 조회
- A5 전표 출력
- 입금 확인

전표 저장 시 난 묶음 예약 수량을 반영한다. 실제 수량 차감은 출고 완료 시 처리한다.
전표 취소 시에는 예약 또는 출고 차감을 역처리한다.

## 6. 경매 추적

경매장 출하 물량을 lot 단위로 추적한다.

주요 기능:

- 경매 lot 조회
- 경매 상태 변경
- 유찰/재경매 대기/반환 확인
- 수량 보정
- 경매 결과 행 관리
- 경매장별 정산 생성
- 정산 재계산

핵심은 행 합계가 아니라 **출하 lot별 상태 추적**이다.

## 7. 입금·정산

거래처별 돈 흐름을 관리한다.

주요 기능:

- 판매 전표 입금 확인
- 경매 정산 입금 확인
- 부분입금 처리
- 거래처 잔액 요약
- 입금 이벤트 목록 조회
- 거래처별 정산 설정

## 8. 품종/입고/자재 관리

품종 기준 정보와 입고 흐름, 자재 기준 정보를 함께 관리한다.

주요 기능:

- 품종 등록/수정/비활성화/삭제
- 품종 중복 방지는 속과 품종명 조합을 기준으로 처리
- 품종별 연결 난 묶음 조회
- 입고 등록
- 입고 상세 수정
- 유리병 모종 포트 작업 등록
- 입고 취소
- 취소된 입고 삭제
- 자재 등록/수정/비활성화/삭제
- 품종/자재/입고 목록 서버 페이지네이션

운영 원칙:

- 품종과 자재는 삭제보다 비활성화를 우선한다.
- 품종 삭제는 연결 난 묶음, 입고 기록이 없을 때만 허용한다.
- 자재 삭제는 연결 데이터가 없을 때만 허용한다.
- 입고 기록은 난 묶음 생성 전까지만 취소 가능하다.
- 취소된 입고는 삭제 가능하고, 자동 생성된 입고 작업 이력은 함께 정리한다.
- 유리병 모종과 즉시 배치 입고는 입력 필드와 후속 흐름이 다르다.

## 9. 분석

농장 운영 데이터를 바탕으로 판단을 돕는 화면이다.

분석 후보:

- 매출/출하 추이
- 품종별 판매 성과
- 거래처별 매출/미입금
- 동/배드/구역 사용률
- 작업 누락 구역
- 상태 이상 추이
- 경매장별 유찰률/반환률/평균 단가
