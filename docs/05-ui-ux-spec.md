# UI/UX 정의서 v2.2

## 1. 기본 원칙

- 모바일보다는 PC/태블릿 우선으로 만든다.
- 글자 크기는 작게 하지 않는다.
- 현장 사용을 고려해 버튼을 크게 만든다.
- 키보드 입력보다 마우스/터치 조작을 우선한다.
- 색상은 흰색, 연한 회색, 초록색 중심으로 한다.
- 선택된 동은 초록색 또는 파란색 테두리로 강조한다.
- 선택된 물리 배드와 논리 구역은 파란색 또는 진한 초록색으로 강조한다.
- 위험/이상 상태는 주황색 또는 빨간색으로 표시한다.
- 화면이 복잡해지면 안 된다.
- 입력 폼은 한 화면에 너무 많은 필드를 넣지 않는다.
- 자주 쓰는 작업은 버튼 클릭 수를 최소화한다.
- 조회 화면과 작업 화면을 분리한다.
- 데이터 변경 작업에는 확인 또는 되돌리기 장치를 둔다.

---

## 2. 화면 역할 분리

### 농장 현황

농장 현황은 통합 조회 화면이다.

```text
목적: 전체 농장 상태 확인 + 동/배드/구역 단위 조회
성격: 보는 화면
주요 조작: 클릭, 줌, 필터, 목록 확인
데이터 변경: 직접 하지 않음
```

농장 현황에서는 전체 농장맵을 중심으로 동, 물리 배드, 논리 구역 단위로 점진적으로 조회한다.

### 난 묶음 관리

난 묶음 관리는 상세 작업 화면이다.

```text
목적: 난 묶음 생성, 수정, 삭제, 이동
성격: 고치는 화면
주요 조작: 구역 클릭, 난 묶음 블록 선택, 드래그 앤 드롭, 목적지 선택
데이터 변경: 가능
```

이 구분을 통해 전체 농장맵이 과도하게 복잡해지는 것을 막고, 실수로 난 묶음을 이동하거나 수정하는 문제를 줄인다.

## 3. 전체 레이아웃

### 좌측 메뉴

```text
대시보드
농장 현황
난 묶음 관리
작업 이력
판매 관리
출력
품종/자재 관리
설정
```

### 대시보드 화면

- 상단 요약 카드
- 최근 작업 요약
- 상태 이상 요약
- 판매 요약
- 빠른 작업 바로가기
- 농장 현황 바로가기
- 난 묶음 관리 바로가기

### 농장 현황 화면

- 중앙 전체 농장맵
- 우측 선택 대상 상세 패널
- 하단 또는 우측 선택 대상의 난 묶음 목록
- 줌 컨트롤

### 난 묶음 관리 화면

- 동 선택 영역
- 선택한 동 상세맵
- 배치 수정 모드 버튼
- 논리 구역별 난 묶음 블록
- 선택한 난 묶음 상세 패널
- 다른 위치로 이동 패널

## 4. 농장 현황 맵 UI

`FarmStatusMap` 컴포넌트를 만든다.

### Props 예시

```ts
type FarmStatusMapProps = {
  houses: HouseStatusSummary[];
  zoomLevel: 'FARM' | 'HOUSE' | 'PHYSICAL_BED' | 'BED_ZONE';
  selectedTarget?: {
    type: 'HOUSE' | 'PHYSICAL_BED' | 'BED_ZONE';
    id: number;
  };
  onSelectHouse: (houseId: number) => void;
  onSelectPhysicalBed: (physicalBedId: number) => void;
  onSelectBedZone: (bedZoneId: number) => void;
  onZoomChange: (zoomLevel: FarmZoomLevel) => void;
};
```

### 구조

```text
FarmStatusMap
 ├─ FarmOverviewLayer
 │   └─ HouseSummaryBlock x 15
 ├─ HouseLayer
 │   └─ PhysicalBedSummaryBlock x 3
 ├─ BedZoneLayer
 │   └─ BedZoneSummaryBlock
 └─ ZoomControl
```

### 전체 농장 보기

전체 농장 보기에서는 15개 동의 상태와 위치를 보여준다. 난 묶음 블록을 세밀하게 표시하지 않는다.

```text
┌────┐ ┌────┐ ┌────┐ ┌────┐ ... ┌────┐
│1동 │ │2동 │ │3동 │ │4동 │     │15동│
└────┘ └────┘ └────┘ └────┘     └────┘
```

동 블록에 표시할 정보:

- 동 번호
- 정상/주의/이상 상태
- 상태 이상 개수
- 분갈이 예정 개수
- 최근 작업 상태

### 클릭/줌 동작

```text
전체 농장 보기에서 동 클릭
→ 해당 동 선택
→ 우측 패널에 해당 동의 모든 난 묶음 목록 표시

줌인
→ 물리 배드가 보이는 단계
→ 배드 클릭
→ 우측 패널에 해당 배드의 모든 난 묶음 목록 표시

더 줌인
→ 논리 구역이 보이는 단계
→ 논리 구역 클릭
→ 우측 패널에 해당 구역의 난 묶음 목록 표시
```

### 선택 대상 패널

선택 대상에 따라 패널 제목과 목록 범위를 바꾼다.

```text
1동 선택:
1동 전체 난 묶음 목록

1동 2배드 선택:
1동 2배드 난 묶음 목록

1동 2배드 우측 구역 선택:
1동 2배드 우측 구역 난 묶음 목록
```

농장 현황 화면의 목록에는 `난 묶음 추가`, `삭제`, `드래그 이동` 같은 변경 버튼을 기본 노출하지 않는다. 필요한 경우 `난 묶음 관리에서 수정` 버튼을 제공한다.

3단계 보강 구현 기준:

- 지도 컨트롤의 확대/축소 버튼은 `전체 → 동 → 배드 → 구역` 단계 표시를 변경한다.
- 전체 보기에서 동을 클릭하면 선택 동의 상세 맵으로 전환한다.
- 동 상세 맵은 3개 물리 배드를 좌→우로 표시하고, 각 배드는 좌/우 논리 구역을 가진다.
- 물리 배드 클릭 시 해당 배드의 난 묶음 목록을 우측 패널과 하단 목록에 표시한다.
- 논리 구역 클릭 시 해당 구역의 난 묶음 목록을 표시한다.
- 농장 현황 맵에서는 데이터 변경 버튼을 제공하지 않고, 수정은 난 묶음 관리 화면으로 이동시킨다.

## 5. 난 묶음 관리 맵 UI

`OrchidManagementMap` 컴포넌트를 만든다.

### Props 예시

```ts
type OrchidManagementMapProps = {
  house: HouseDetail;
  selectedPhysicalBedId?: number;
  selectedBedZoneId?: number;
  selectedOrchidGroupId?: number;
  editMode: boolean;
  viewMode: 'REAL_DIRECTION' | 'ROTATED' | 'BY_BED';
  onSelectPhysicalBed: (physicalBedId: number) => void;
  onSelectBedZone: (bedZoneId: number) => void;
  onSelectOrchidGroup: (orchidGroupId: number) => void;
  onMoveWithinHouse: (orchidGroupId: number, toBedZoneId: number) => void;
  onOpenMoveDestinationPicker: (orchidGroupId: number) => void;
};
```

### 구조

```text
OrchidManagementMap
 ├─ HouseDetailHeader
 ├─ ViewModeToggle
 │   ├─ 실제 방향 보기
 │   ├─ 회전 보기
 │   └─ 배드별 보기
 ├─ PhysicalBedBlock 1
 │   ├─ BedZone 좌
 │   └─ BedZone 우
 ├─ PhysicalBedBlock 2
 │   ├─ BedZone 좌
 │   └─ BedZone 우
 ├─ PhysicalBedBlock 3
 │   ├─ BedZone 좌
 │   └─ BedZone 우
 └─ EditModeControl
```

### 실제 방향 보기

한 동 내부의 물리 배드는 실제 농장 구조와 동일하게 왼쪽에서 오른쪽으로 표시한다.

```text
┌────────────────────────────────────┐
│ 1동                                 │
│ ┌────────┐ ┌────────┐ ┌────────┐   │
│ │ 1배드   │ │ 2배드   │ │ 3배드   │   │
│ │ 좌 | 우 │ │ 좌 | 우 │ │ 좌 | 우 │   │
│ └────────┘ └────────┘ └────────┘   │
└────────────────────────────────────┘
```

논리 구역 내부는 세로로 긴 영역으로 표현한다. 난 묶음은 다음 두 방식 중 하나로 표현할 수 있다.

```text
규격 격자형: 비슷한 크기의 블록을 반복 배치
덩어리형: 실제 난 묶음 규모에 맞춰 큰 블록으로 배치
```

MVP에서는 덩어리형을 우선 고려한다. 실제 농장에서는 난 묶음의 크기가 균일하지 않을 수 있기 때문이다.

### 보기 옵션

동 상세 보기의 가로 공간이 부족할 수 있으므로 다음 보기 옵션을 제공할 수 있다.

```text
[실제 방향 보기] [회전 보기] [배드별 보기]
```

- 실제 방향 보기: 실제 농장 방향 그대로 1배드, 2배드, 3배드를 좌→우로 표시한다.
- 회전 보기: 조작성 확보를 위해 동 상세 화면만 90도 회전해 표시한다.
- 배드별 보기: 1배드/2배드/3배드를 탭 또는 컬럼으로 분리해 표시한다.

데이터 기준 방향은 항상 실제 농장 방향을 따른다. 회전 보기는 화면 렌더링 방식일 뿐 데이터의 배치 순서를 변경하지 않는다.

### 색상 기본값

- 초록색: 난 묶음이 배치된 영역
- 파란색: 선택된 영역 또는 선택된 난 묶음
- 회색: 빈 영역
- 주황색: 주의 상태
- 빨간색: 이상 상태

## 6. 보기 모드와 배치 수정 모드

### 보기 모드

기본 상태다.

- 동/물리 배드/논리 구역 선택 가능
- 난 묶음 상세 조회 가능
- 작업 기록 추가 가능
- 난 묶음 드래그 이동 불가

### 배치 수정 모드

사용자가 명시적으로 켜야 하는 모드다.

- 같은 동 내부 난 묶음 블록 드래그 가능
- 같은 동 내부 다른 논리 구역으로 드롭 가능
- 드롭 후 확인 창 표시
- 확인 시 이동 처리
- 이동 이력 자동 생성
- 이동 직후 되돌리기 표시 권장

배치 수정 모드 버튼 예시:

```text
[배치 수정 시작]
[배치 수정 종료]
```

다른 동으로 이동할 때는 드래그 앤 드롭이 아니라 `다른 위치로 이동` 버튼을 사용한다.

## 7. 난 묶음 생성 UX

### 기본 흐름

```text
난 묶음 관리 화면 진입
→ 동 선택
→ 논리 구역 클릭
→ [난 묶음 추가]
→ 품종명 입력 또는 선택
→ 수량 입력
→ 상태 선택
→ 저장
```

### 입력 필드 우선순위

MVP에서 필수에 가까운 필드:

```text
품종명
수량
상태
```

선택 필드:

```text
속
화분 크기
연차
메모
```

부모님 세대 사용성을 고려해 긴 텍스트 입력은 최소화한다. 상태, 속, 화분 크기 등은 가능하면 선택 버튼 또는 드롭다운으로 제공한다.

## 8. 난 묶음 이동 UX

### 같은 동 내부 이동

```text
배치 수정 모드 켜기
→ 난 묶음 블록을 마우스로 잡기
→ 같은 동 안의 다른 논리 구역으로 드래그
→ 드롭
→ 이동 확인 창
→ 확인
→ 이동 완료
```

### 다른 동으로 이동

```text
난 묶음 블록 선택
→ [다른 위치로 이동]
→ 목적 동 선택
→ 목적 물리 배드 선택
→ 목적 논리 구역 선택
→ 이동 확인 창
→ 확인
→ 이동 완료
```

목적지 선택 패널 예시:

```text
다른 위치로 이동

선택한 난 묶음: 카틀레야 A / 120분
현재 위치: 1동 2배드 우측

목적 동: [5동 ▼]
목적 배드: [1배드 ▼]
목적 구역: [우측 ▼]

[이동] [취소]
```

### 확인 창

```text
카틀레야 A / 120분을 이동할까요?

이전 위치: 1동 2배드 우측
새 위치: 5동 1배드 우측

[이동] [취소]
```

동 간 이동 확인 창에서는 이전 동과 새 동을 반드시 표시한다.

### 되돌리기

이동 완료 후 화면 하단에 짧은 알림을 표시한다.

```text
카틀레야 A를 5동 1배드 우측으로 이동했습니다. [되돌리기]
```

### 일부 이동

MVP에서는 일부 이동을 지원하지 않는다. MVP 이후 다음 흐름으로 확장한다.

```text
드롭 또는 이동 버튼 클릭
→ 전체 이동 / 일부 이동 선택
→ 일부 이동 선택 시 이동 수량 입력
→ 기존 난 묶음 수량 차감 + 새 난 묶음 생성
```

## 9. 작업 이력 입력 UX

작업 이력 입력은 다음 흐름을 따른다.

```text
작업 기록 추가 클릭
→ 작업 유형 선택
→ 대상 자동 선택 또는 변경
→ 날짜 입력, 기본값 오늘
→ 작업 유형별 추가 필드 입력
→ 저장
```

작업 유형에 따라 불필요한 필드는 숨긴다.

예시:

- 농약: 약제명, 희석 배수 표시
- 비료: 비료명, 희석 배수 표시
- 분갈이: 화분 크기, 식재 재료, 메모 표시
- 잎 정리: 메모 중심
- 잡초 정리: 메모 중심
- 단화/꽃 정리: 메모 중심
- 일반 메모: 메모만 표시
- 위치 이동: 시스템 자동 생성, 필요 시 메모만 추가

---

## 10. 판매 전표 입력 UX

판매 전표 입력은 다음 흐름을 따른다.

```text
판매 전표 등록
→ 거래처 선택 또는 신규 등록
→ 판매 품목 추가
→ 수량/단가 입력
→ 금액 자동 계산
→ 저장
→ A5 출력
```

MVP에서는 판매 품목을 자유 입력한다.

MVP 이후에는 판매 품목 입력 시 난 묶음을 검색하거나 농장맵에서 선택할 수 있도록 확장한다.

## 11. 출력 UX

출력물은 부모님 세대 사용자가 읽기 쉬운 것을 최우선으로 한다.

- A5 형식 기본 사용
- 표 중심 구성
- 큰 글자
- 불필요한 색상 최소화
- 출력 화면에서는 좌측 메뉴, 상단 네비게이션을 숨긴다.
- A5 한 장에 너무 많은 정보를 넣지 않는다.

출력 대상별 권장 방식:

| 출력물 | 권장 방식 |
|---|---|
| 판매 전표 | A5 1장 기준 |
| 논리 구역 관리표 | A5 1장 또는 여러 장 |
| 물리 배드 관리표 | A5 여러 장 허용 |
