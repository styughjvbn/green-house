# 기능정의서 v2.2

## 1. 대시보드

### 기능 설명

농장 전체 운영 상태를 요약해서 확인하는 화면이다. 대시보드는 상세 조작 화면이 아니라, 오늘 확인해야 할 주요 지표와 최근 작업을 빠르게 보는 진입 화면이다.

### 표시 정보

- 전체 동 수: 15개
- 전체 물리 배드 수: 45개
- 전체 기본 논리 구역 수: 90개
- 전체 난 묶음 수
- 분갈이 예정 개수
- 상태 이상 개수
- 최근 농약 살포일
- 최근 비료 작업일
- 최근 분갈이 작업일
- 최근 위치 이동 이력
- 최근 작업 이력
- 오늘 확인 필요 항목
- 15동 상태 요약
- 판매 요약
- 빠른 작업 링크

### 주요 동작

- 사용자는 대시보드에서 농장 현황 화면으로 이동할 수 있다.
- 사용자는 대시보드에서 난 묶음 관리 화면으로 이동할 수 있다.
- 사용자는 대시보드에서 작업 이력, 판매 관리, 출력 화면으로 빠르게 이동할 수 있다.

## 2. 농장 현황 화면

### 기능 설명

농장 현황 화면은 전체 농장을 통합 조회하는 화면이다. 이 화면의 핵심 목적은 난 묶음을 직접 이동하거나 수정하는 것이 아니라, 전체 동/배드/구역의 상태와 난 묶음 분포를 한눈에 파악하는 것이다.

### 핵심 역할

```text
농장 현황 = 전체 보기 + 줌 단계별 조회 + 선택 범위의 난 묶음 목록 확인
```

### 표시 정보

- 전체 농장맵
- 15개 동의 위치 및 상태
- 동별 상태 요약
- 동별 상태 이상 개수
- 동별 분갈이 예정 개수
- 최근 작업일 요약
- 선택한 대상의 난 묶음 목록

### 선택 단계

농장 현황 화면은 줌 단계에 따라 선택 대상을 다르게 한다.

```text
전체 농장 보기
→ 동 선택
→ 물리 배드가 보이는 단계
→ 물리 배드 선택
→ 논리 구역이 보이는 단계
→ 논리 구역 선택
```

### 선택 시 우측 패널 표시 규칙

#### 동 선택 시

동을 클릭하면 해당 동에 존재하는 모든 난 묶음 목록을 표시한다.

```text
1동 전체 난 묶음
- 1배드 좌 / 카틀레야 A / 120분 / 정상
- 1배드 우 / 덴드로비움 B / 80분 / 개화중
- 2배드 좌 / 심비디움 C / 200분 / 판매 가능
```

#### 물리 배드 선택 시

줌인하여 물리 배드가 보이는 단계에서 배드를 클릭하면 해당 배드에 존재하는 모든 난 묶음 목록을 표시한다.

```text
1동 - 2배드 난 묶음
- 좌측 구역 / 카틀레야 A / 120분
- 우측 구역 / 덴드로비움 C / 80분
```

#### 논리 구역 선택 시

논리 구역을 클릭하면 해당 구역의 난 묶음 목록과 최근 작업 요약을 표시한다.

```text
1동 - 2배드 우측 구역
- 덴드로비움 C / 80분

최근 농약: 2026-06-01
최근 비료: 2026-05-20
최근 분갈이: 2026-03-12
상태: 정상
```

### 농장 현황 화면에서 하지 않는 일

다음 작업은 농장 현황 화면에서 직접 수행하지 않는다.

```text
난 묶음 생성
난 묶음 수정
난 묶음 삭제
난 묶음 드래그 이동
논리 구역 배치 수정
```

이 작업들은 난 묶음 관리 화면에서 수행한다.

## 3. 농장맵

### 기능 설명

농장맵은 실제 농장 구조를 참고한 관리용 지도다. 정밀 CAD 도면이나 GIS 지도가 아니라, 현장에서 위치를 이해하고 빠르게 조회하기 위한 UI다.

### 전체 보기 요구사항

- 15개 동을 화면에 배치한다.
- 각 동은 상태 확인과 선택을 위한 지도 블록으로 표현한다.
- 각 동에는 `1동`, `2동`, ..., `15동` 라벨을 붙인다.
- 동별 상태를 색상 또는 배지로 표시한다.
- 전체 보기에서는 난 묶음 블록을 세밀하게 표시하지 않는다.
- 동을 클릭하면 해당 동이 선택되고, 우측 패널에는 동 전체 난 묶음 목록이 표시된다.
- 줌인하면 물리 배드가 보이는 단계로 진입한다.

### 동 상세 구조 요구사항

동 상세 구조는 난 묶음 관리 화면에서 주로 사용한다.

- 선택한 동 1개를 크게 표시한다.
- 한 동 안에는 3개의 물리 배드를 표시한다.
- 3개의 물리 배드는 실제 농장과 동일하게 왼쪽에서 오른쪽으로 `1배드`, `2배드`, `3배드` 순서로 표시한다.
- 각 물리 배드는 좌/우 2개의 논리 구역으로 나누어 표시한다.
- 논리 구역 안에는 난 묶음 블록을 표시한다.

### 맵 구조 예시

전체 농장 보기:

```text
┌────┐ ┌────┐ ┌────┐ ┌────┐ ... ┌────┐
│1동 │ │2동 │ │3동 │ │4동 │     │15동│
└────┘ └────┘ └────┘ └────┘     └────┘
```

동 상세 보기:

```text
┌────────────────────────────────────┐
│ 1동                                 │
│ ┌────────┐ ┌────────┐ ┌────────┐   │
│ │ 1배드   │ │ 2배드   │ │ 3배드   │   │
│ │ 좌 | 우 │ │ 좌 | 우 │ │ 좌 | 우 │   │
│ └────────┘ └────────┘ └────────┘   │
└────────────────────────────────────┘
```

### 줌 개념

```text
전체 농장 보기
→ 동 보기
→ 물리 배드 보기
→ 논리 구역 보기
→ 세부 그리드 보기
→ 개별 화분 보기
```

MVP에서는 정밀 좌표 기반 편집까지 구현하지 않는다. 다만 줌 단계별 조회 흐름은 고려한다.

## 4. 난 묶음 관리 화면

### 기능 설명

난 묶음 관리 화면은 선택한 동 또는 구역 안에서 실제 난 묶음을 관리하는 작업 화면이다. 농장 현황 화면이 조회 중심이라면, 난 묶음 관리 화면은 생성, 수정, 삭제, 이동 중심이다.

### 핵심 역할

```text
난 묶음 관리 = 동 상세맵 + 난 묶음 블록 + 배치 수정 + 위치 이동
```

### 기능 목록

- 동 선택
- 선택한 동의 상세맵 표시
- 물리 배드 1, 2, 3을 좌→우로 표시
- 각 물리 배드의 좌/우 논리 구역 표시
- 논리 구역별 난 묶음 목록 표시
- 난 묶음 등록
- 난 묶음 수정
- 난 묶음 삭제
- 같은 동 내부 난 묶음 드래그 이동
- 다른 동으로 이동할 때 목적지 선택 방식 제공
- 난 묶음 구역 내 순서 관리

### 난 묶음 필드

```text
id
동 번호
물리 배드 번호
논리 구역 id
속
품종명
수량
화분 크기
연차
상태
표시 순서
메모
등록일
수정일
```

### 난 묶음 블록 UI

```text
[카틀레야 A / 120분 / 정상]
```

블록에는 최소한 다음 정보를 표시한다.

- 품종명 또는 상품명
- 수량
- 상태

색상 기본값:

- 초록색: 난 묶음이 배치된 영역
- 파란색: 선택된 영역 또는 선택된 난 묶음
- 회색: 빈 영역
- 주황/빨강: 주의 또는 이상 상태

## 5. 맵 기반 난 묶음 생성

### 기능 설명

난 묶음 관리 화면에서 논리 구역을 선택한 뒤 난 묶음을 생성한다.

### MVP 흐름

```text
난 묶음 관리 화면 진입
→ 동 선택
→ 논리 구역 클릭
→ [난 묶음 추가] 클릭
→ 품종명, 수량, 상태 등 최소 정보 입력
→ 저장
→ 해당 구역에 난 묶음 블록 생성
```

### MVP 제외

```text
마우스 드래그로 자유 영역 생성
구역 병합/분할
철사/판 단위 위치 지정
개별 화분 위치 지정
```

## 6. 맵 기반 난 묶음 이동

### 기능 설명

난 묶음 관리 화면에서 난 묶음을 다른 논리 구역으로 이동한다. 이동 방식은 같은 동 내부 이동과 다른 동 이동으로 나눈다.

### 같은 동 내부 이동

같은 동 안에서는 난 묶음 블록을 드래그 앤 드롭하여 이동한다.

```text
[배치 수정] 모드 켜기
→ 난 묶음 블록 드래그
→ 같은 동 안의 대상 논리 구역에 드롭
→ 이동 확인 창 표시
→ 확인
→ 난 묶음의 논리 구역 변경
→ 위치 이동 작업 이력 자동 생성
```

### 다른 동으로 이동

다른 동으로 이동할 때는 드래그만으로 처리하지 않는다. 목적지를 단계적으로 선택한다.

```text
난 묶음 블록 선택
→ [다른 위치로 이동] 클릭
→ 목적 동 선택
→ 목적 물리 배드 선택
→ 목적 논리 구역 선택
→ 이동 확인 창 표시
→ 확인
→ 난 묶음의 논리 구역 변경
→ 위치 이동 작업 이력 자동 생성
```

### 확인 창 예시

```text
카틀레야 A / 120분을 이동할까요?

이전 위치: 3동 2배드 좌측
새 위치: 5동 1배드 우측

[이동] [취소]
```

### MVP 규칙

- 기본 모드는 보기 모드다.
- 배치 수정 모드에서만 같은 동 내부 드래그 이동이 가능하다.
- 다른 동 이동은 목적지 선택 방식으로 처리한다.
- MVP에서는 난 묶음 전체 이동만 지원한다.
- 이동 시 위치 이동 작업 이력을 자동 생성한다.
- 이동 직후 되돌리기 기능을 제공하는 것이 권장된다.

### MVP 이후 확장

- 일부 수량 이동
- 난 묶음 자동 분할
- 여러 난 묶음 일괄 이동
- 이동 바구니
- 구역 병합/분할과 연동

## 7. 작업 이력 관리

### 기능 설명

농약, 비료, 분갈이, 정리 작업 등의 작업 이력을 기록한다.

### 작업 대상

```text
전체 농장
특정 동
특정 물리 배드
특정 논리 구역
특정 난 묶음
```

### MVP 작업 유형

```text
농약
비료
분갈이
상태 기록
일반 메모
잎 정리
잡초 정리
단화 / 꽃 정리
위치 이동
```

### 주요 규칙

- 최근 농약일, 최근 비료일, 최근 분갈이일은 작업 이력에서 자동 계산한다.
- 사용자가 최근 작업일을 직접 수정하지 않는다.
- 난 묶음 이동 시 위치 이동 작업 이력을 자동 생성한다.
- MVP 이후에는 사용자가 작업 유형을 추가할 수 있어야 한다.

## 8. 판매 전표 관리

### 기능 설명

기존 수기 전표를 디지털화하기 위한 기능이다.

### 기능 목록

- 거래처 등록
- 판매 전표 생성
- 판매 품목 추가
- 수량/단가/금액 자동 계산
- 입금 여부 관리
- 메모 작성
- A5 전표 출력
- 거래처별 판매 내역 조회
- 월별 매출 조회

### MVP 규칙

- MVP에서는 자유 입력 기반 전표를 생성한다.
- 판매 품목은 반드시 난 묶음과 연결되지 않아도 된다.
- 전표 생성과 출력이 핵심이다.

### MVP 이후 규칙

- 판매 품목을 난 묶음과 연결할 수 있다.
- 판매 확정 또는 출고 완료 시 난 묶음 수량을 자동 차감한다.
- 전표 취소 또는 출고 취소 시 수량 복구 정책이 필요하다.

---

## 9. 출력 기능

### 출력 대상

```text
판매 전표 출력
거래처별 월 정산표 출력
논리 구역별 관리표 출력
물리 배드별 관리표 출력
작업 이력 출력
```

MVP 구현 범위는 판매 전표 A5 출력까지다. 거래처별 월 정산표, 논리 구역별 관리표, 물리 배드별 관리표, 작업 이력 출력은 이후 범위로 둔다.

### 출력 원칙

- A5 기준으로 읽기 쉽게 만든다.
- 글자를 작게 만들지 않는다.
- A5 한 장에 너무 많은 정보를 넣지 않는다.
- 복잡한 디자인보다 정보의 명확성을 우선한다.
- 브라우저 print CSS를 적용한다.
- 출력 화면에서는 좌측 메뉴, 상단 네비게이션을 숨긴다.
- 필요하면 A4 출력은 MVP 이후 옵션으로 제공한다.

## 13. 8단계 판매 전표 관리 구현 범위

판매 관리 화면에서 거래처를 등록하고 판매 전표를 생성/조회할 수 있다. 판매 품목은 자유 입력 기반이며, 수량과 단가로 품목 금액 및 전표 합계를 자동 계산한다.

이번 단계에서 제공하는 작업:

- 거래처 등록
- 거래처 목록 조회
- 판매 전표 생성
- 판매 전표 목록 조회
- 판매 전표 상세 조회 API
- 판매 품목 금액 자동 계산
- 전표 합계 자동 계산

이번 단계에서 제외하는 작업:

- 판매 전표 수정/삭제
- 판매 상태 변경 API
- 난 묶음 수량 자동 차감
- 판매 품목과 난 묶음 검색 연결 고도화
- A5 전표 출력

A5 출력과 인쇄 CSS는 9단계 출력 기능에서 구현한다. 판매 수량 자동 차감은 MVP 범위가 아니므로 임의로 적용하지 않는다.

---

## 14. 9단계 판매 전표 A5 출력 구현 범위

출력 화면에서 판매 전표를 선택하고 A5 기준 출력 화면으로 이동할 수 있다. 판매 전표 출력 화면은 브라우저 인쇄 기능을 사용하며, 인쇄 시 좌측 메뉴와 화면용 버튼을 숨긴다.

이번 단계에서 제공하는 작업:

- 판매 전표 출력 데이터 조회 API
- 출력 메뉴의 판매 전표 목록
- 판매 관리 목록의 A5 출력 링크
- 판매 전표 A5 화면
- 브라우저 print CSS

이번 단계에서 제외하는 작업:

- 논리 구역별 관리표 출력
- 물리 배드별 관리표 출력
- 작업 이력 출력
- PDF 파일 생성/저장
- 판매 수량 자동 차감

구역/배드 관리표와 작업 이력 출력은 다음 출력 범위에서 구현한다.

---

## 12. 7단계 작업 이력 관리 구현 범위

작업 이력 화면에서 농약, 비료, 분갈이, 상태 기록, 일반 메모, 잎 정리, 잡초 정리, 단화/꽃 정리 작업을 등록하고 조회할 수 있다.

이번 단계에서 제공하는 작업:

- 작업 이력 등록
- 작업 대상 유형 선택: 전체 농장, 동, 물리 배드, 논리 구역, 난 묶음
- 작업 대상 목록 선택
- 작업일, 작업 유형, 자재명, 희석 배수, 수량, 작업자, 메모 입력
- 최근 작업 이력 목록 조회
- 작업 유형 기본 목록 제공

이번 단계에서 제외하는 작업:

- 작업 이력 수정
- 작업 이력 삭제
- 작업 유형 사용자 추가/수정
- 최근 농약/비료/분갈이 날짜의 대시보드 자동 반영 고도화
- 작업 이력 A5 출력

난 묶음 위치 이동으로 자동 생성되는 `위치 이동` 이력은 계속 시스템 자동 생성 흐름으로 유지한다.

난 묶음 관리 화면에서는 선택한 난 묶음 또는 구역 기준으로 작업 기록을 바로 추가할 수 있다. 이 기능은 기존 작업 이력 API를 재사용하며, 저장 후 현재 화면을 재조회한다.

---

## 11. 6단계 난 묶음 위치 이동 구현 범위

난 묶음 관리 화면에서 선택한 난 묶음 전체를 다른 논리 구역으로 이동할 수 있다. 목적지는 동, 물리 배드, 논리 구역 순서로 선택한다.

이번 단계에서 제공하는 작업:

- 난 묶음 선택 후 다른 위치로 이동 패널 열기
- 목적 동/배드/구역 선택
- 같은 동 또는 다른 동의 논리 구역으로 난 묶음 전체 이동
- 배치 수정 모드에서 같은 동 내부 드래그 앤 드롭 이동
- 드롭 전 이동 확인
- 이동 성공 후 현재 화면 재조회
- 이동 시 `위치 이동` 작업 이력 자동 생성

이번 단계에서 제외하는 작업:

- 일부 수량 이동
- 난 묶음 자동 분할
- 여러 난 묶음 일괄 이동
- 이동 직후 되돌리기

이동 직후 되돌리기는 MVP 이후 개선 항목으로 남긴다.

---

## 10. 5단계 난 묶음 CRUD 구현 범위

난 묶음 관리 화면에서 선택한 논리 구역 기준으로 난 묶음을 생성할 수 있다. 난 묶음을 선택하면 품종명, 수량, 상태, 화분 크기, 년생, 배치 유형, 트레이 수, 메모를 수정할 수 있다.

이번 단계에서 제공하는 작업:

- 논리 구역 선택 후 난 묶음 추가
- 난 묶음 선택 후 상세 정보 수정
- 난 묶음 삭제
- 저장 후 현재 동 상세맵 재조회

이번 단계에서 제외하는 작업:

- 난 묶음 위치 이동
- 드래그 앤 드롭
- 이동 작업 이력 자동 생성
- 일부 수량 이동 또는 난 묶음 분할

위치 이동은 6단계에서 구현하며, 이동 시 작업 이력 자동 생성 규칙을 함께 적용한다.
