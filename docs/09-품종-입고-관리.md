# 품종/입고 기능 최종 설계안

## 1. 메뉴 구조

```text
품종/자재 관리
├─ 품종
├─ 입고
└─ 자재
```

`품종/자재 관리`는 기존 메뉴를 유지한다.
기존 문서의 품종/자재 관리 화면은 품종 목록, 품종 상세, 연결된 난 묶음, 자재 목록을 다루는 구조이므로 여기에 `입고` 탭을 추가한다. 

---

# 2. 품종 탭

## 2.1 역할

```text
품종 기준 정보 관리
품종별 보유 난 묶음 확인
품종별 총 보유 수량 확인
품종 중복/비활성 관리
```

## 2.2 주요 기능

```text
품종 목록 조회
품종 등록
품종 수정
품종 비활성화
품종 검색/필터
선택 품종 상세 조회
선택 품종의 연결 난 묶음 조회
품종별 총 보유 수량 조회
```


# 3. 입고 탭

## 3.1 역할

```text
외부에서 농장으로 들어온 모종/상품분/샘플 기록
새 품종 생성 또는 기존 품종 연결
유리병 모종 임시보관 관리
포트묘/상품분의 초기 난 묶음 생성
입고 작업 이력 생성
```

입고는 외부 유입만 처리한다.

```text
포함:
- 유리병 모종
- 포트 모종
- 상품분
- 샘플
- 기타 외부 유입

제외:
- 기존 난 분주
- 위치 이동
- 분갈이
- 폐기
- 경매장 반환
```

분주는 `난 묶음 관리`에서 처리한다.

# 10. 난 묶음 생성 규칙

난 묶음은 실제 배드에 올라간 시점에 생성한다.

```text
유리병 모종 입고:
- 입고 시점에는 난 묶음 생성 안 함
- 포트 작업 후 난 묶음 생성

포트 모종 / 상품분 / 샘플:
- 입고 저장 시 난 묶음 생성 가능
```

난 묶음은 기존 시스템 규칙대로 논리 구역에 배치한다. 현재 도메인 구조상 난 묶음은 하나의 논리 구역에 속하고, 수량·화분 크기·연차·상태·표시 순서를 가진다. 

---

# 11. 분주 처리 위치

분주는 입고 탭에서 처리하지 않는다.

```text
난 묶음 관리
→ 기존 난 묶음 선택
→ 분주하기
→ 원본 수량 조정 또는 종료
→ 새 난 묶음 생성
→ 분주 작업 이력 생성
```

분주 결과로 생긴 새 묶음은 입고 기록이 아니라 `분주 이력`을 가진다.

```text

```

## 12.2 자동 생성 이력

```text
입고 저장
→ 입고 작업 이력 생성

→ 포트 작업 또는 분갈이 작업 이력 생성
```

작업 이력은 기존 구조대로 전체 농장, 동, 물리 배드, 논리 구역, 난 묶음 중 하나를 대상으로 기록할 수 있다.

# 14. 관계

```text
Variety 1 ─ N OrchidGroup
Variety 1 ─ N InboundRecord
InboundRecord 0..1 ─ 1 OrchidGroup
BedZone 1 ─ N OrchidGroup
BedZone 1 ─ N InboundRecord
```

유리병 모종 입고의 경우:

```text
InboundRecord.created_orchid_group_id = null
InboundRecord.bed_zone_id = null
```

포트 작업 완료 후:

```text
InboundRecord.created_orchid_group_id = 생성된 난 묶음 ID
InboundRecord.bed_zone_id = 배치된 논리 구역 ID
```
