# 도메인 모델 문서 v2

## 1. 도메인 개요

난 농장 관리 시스템의 핵심 도메인은 다음과 같다.

- 농장
- 동
- 물리 배드
- 논리 구역
- 난 묶음
- 작업 이력
- 작업 유형
- 거래처
- 판매 전표
- 판매 품목

초기 MVP는 개별 화분 단위가 아니라 난 묶음 단위로 관리한다.

v2에서는 실제 농장의 배치 구조를 반영하기 위해 기존 `동 → 배드 → 난 묶음` 구조를 `동 → 물리 배드 → 논리 구역 → 난 묶음` 구조로 확장한다.

---

## 2. 농장 구조

농장은 다음 구조를 가진다.

```text
농장
 ├─ 1동
 │   ├─ 1번 물리 배드
 │   │   ├─ 좌측 논리 구역
 │   │   └─ 우측 논리 구역
 │   ├─ 2번 물리 배드
 │   │   ├─ 좌측 논리 구역
 │   │   └─ 우측 논리 구역
 │   └─ 3번 물리 배드
 │       ├─ 좌측 논리 구역
 │       └─ 우측 논리 구역
 ...
 └─ 15동
```

규칙:

- 농장은 15개의 동을 가진다.
- 각 동은 3개의 물리 배드를 가진다.
- 전체 물리 배드 수는 45개다.
- 각 물리 배드는 기본적으로 좌/우 2개의 논리 구역을 가진다.
- 전체 기본 논리 구역 수는 90개다.
- 각 논리 구역에는 여러 난 묶음이 존재할 수 있다.
- 행잉 분화는 MVP 이후 별도 논리 구역 유형으로 확장할 수 있다.

---

## 3. 물리 배드

물리 배드는 실제 농장에 존재하는 긴 재배 배드다.

특징:

- 세로로 긴 철사가 나란히 놓여 있다.
- 철사 사이에 분화를 걸 수 있다.
- 20구 또는 15구 판을 끼우고 그 판에 분화를 담을 수 있다.
- 중앙 세로선 또는 관수 라인을 기준으로 좌/우로 나누어 활용되는 경우가 많다.
- 실제 규격은 어느 정도 표준화되어 있으나 현장 오차가 존재한다.

물리 배드는 다음 정보를 가질 수 있다.

```text
동 번호
배드 번호
대략적인 길이
대략적인 폭
철사 줄 수
받침대 간격
메모
```

MVP에서는 물리 배드의 규격 정보를 필수 입력으로 요구하지 않는다. 다만 향후 농장맵 그리드 생성 기준으로 사용할 수 있도록 확장 가능하게 둔다.

---

## 4. 논리 구역

논리 구역은 난 묶음을 배치하고 관리하는 실제 시스템상의 위치 단위다.

MVP에서는 각 물리 배드에 다음 기본 논리 구역을 자동 생성한다.

```text
1번 배드 좌
1번 배드 우
2번 배드 좌
2번 배드 우
3번 배드 좌
3번 배드 우
```

논리 구역은 다음 역할을 가진다.

- 난 묶음이 배치되는 위치 단위
- 작업 이력의 대상 단위
- 농장맵에서 선택 가능한 단위
- 출력물에서 관리표를 나눌 수 있는 단위

MVP 이후에는 다음 기능으로 확장할 수 있다.

- 논리 구역 병합
- 논리 구역 분할
- 논리 구역 이름 변경
- 행잉 구역 추가
- 판 단위 구역 추가
- 개별 화분 위치 관리
- 그리드 좌표 기반 구역 관리

---

## 5. 난 묶음

난 묶음은 초기 MVP의 핵심 관리 단위다.

난 묶음은 다음 조건을 만족하는 단위다.

```text
같은 논리 구역에 위치하고,
같은 품종 또는 같은 관리 그룹이며,
같은 화분 크기/연차/상태로 묶어서 관리할 수 있는 난들
```

예시:

```text
3동 - 2번 배드 좌측 구역
 ├─ 카틀레야 A / 4치 / 120분 / 2년차
 ├─ 카틀레야 B / 5치 / 80분 / 개화주
 └─ 덴드로비움 C / 3.5치 / 200분 / 판매 가능
```

규칙:

- 난 묶음은 하나의 논리 구역에 속한다.
- 하나의 논리 구역에는 여러 난 묶음이 존재할 수 있다.
- 난 묶음은 구역 안에서 표시 순서를 가진다.
- 난 묶음은 다른 논리 구역으로 이동할 수 있다.
- MVP에서는 난 묶음 전체 이동을 지원한다.
- MVP 이후에는 일부 수량 이동과 자동 분할을 지원할 수 있다.

---

## 6. 난 묶음 이동

난 묶음 이동은 단순 위치 변경이 아니라 농장 작업 이력으로 간주한다.

예시:

```text
카틀레야 A / 120분
3동 2배드 좌측 구역 → 5동 1배드 우측 구역
```

MVP 규칙:

- 난 묶음 블록을 드래그 앤 드롭하여 다른 논리 구역으로 이동한다.
- 이동은 전체 수량 이동만 지원한다.
- 이동 전 사용자 확인을 받는다.
- 이동이 완료되면 위치 이동 작업 이력을 자동 생성한다.
- 이동 이력에는 이전 구역, 새 구역, 이동 수량, 작업자, 메모를 남길 수 있다.

MVP 이후 규칙:

- 일부 수량 이동을 지원한다.
- 일부 수량 이동 시 기존 난 묶음은 남은 수량으로 갱신되고, 이동 수량만큼 새 난 묶음이 생성된다.

예시:

```text
기존:
카틀레야 A / 120분 / 3동 2배드 좌

40분 이동 후:
카틀레야 A / 80분 / 3동 2배드 좌
카틀레야 A / 40분 / 5동 1배드 우
```

---

## 7. 작업 이력

작업 이력은 농약, 비료, 분갈이, 정리 작업, 상태 기록, 메모 등의 이벤트를 기록한다.

작업 대상은 다음 중 하나다.

```text
전체 농장
특정 동
특정 물리 배드
특정 논리 구역
특정 난 묶음
```

작업 유형 예시:

```text
농약
비료
분갈이
관수
위치 이동
잎 정리
잡초 정리
단화 / 꽃 정리
병해충 발견
생육 이상
개화 확인
입고
출고
폐기
상태 기록
일반 메모
```

MVP 우선 구현 대상:

```text
농약
비료
분갈이
상태 기록
일반 메모
잎 정리
잡초 정리
단화 / 꽃 정리
위치 이동
```

작업 유형은 MVP에서는 기본값으로 제공한다. MVP 이후에는 사용자가 작업 유형을 직접 추가/수정/비활성화할 수 있도록 코드 테이블로 확장한다.

---

## 8. 최근 작업일 계산 규칙

특정 논리 구역의 최근 농약일을 계산할 때는 다음 이력을 고려한다.

- 해당 논리 구역에 직접 등록된 농약 이력
- 해당 논리 구역이 속한 물리 배드에 등록된 농약 이력
- 해당 물리 배드가 속한 동에 등록된 농약 이력
- 전체 농장에 등록된 농약 이력

특정 난 묶음의 최근 농약일을 계산할 때는 다음 이력을 고려한다.

- 해당 난 묶음에 직접 등록된 농약 이력
- 해당 난 묶음이 속한 논리 구역에 등록된 농약 이력
- 해당 논리 구역이 속한 물리 배드에 등록된 농약 이력
- 해당 물리 배드가 속한 동에 등록된 농약 이력
- 전체 농장에 등록된 농약 이력

비료도 같은 방식으로 계산한다.

분갈이는 보통 난 묶음 또는 논리 구역 단위가 자연스럽다. 다만 시스템 구조상 전체 농장/동/물리 배드/논리 구역/난 묶음 대상 기록을 모두 허용한다.

---

## 9. 판매 도메인

판매 도메인은 거래처, 판매 전표, 판매 품목으로 구성된다.

규칙:

- 하나의 거래처는 여러 판매 전표를 가질 수 있다.
- 하나의 판매 전표는 여러 판매 품목을 가질 수 있다.
- 판매 품목 금액은 수량 × 단가로 계산한다.
- 판매 전표 총 금액은 판매 품목 금액 합계로 계산한다.
- 판매 전표는 A5 기준으로 출력 가능한 형태여야 한다.

MVP에서는 판매 품목을 자유 입력 방식으로 등록한다.

MVP 이후에는 판매 품목이 특정 난 묶음과 연결될 수 있다.

```text
판매 품목 → 난 묶음 nullable 연결
```

판매 수량 자동 차감은 MVP 이후 기능이다. 차감 시점은 전표 생성이 아니라 판매 확정 또는 출고 완료 상태를 기준으로 하는 것이 권장된다.

## 10. 도메인 용어 사전

| 용어 | 의미 |
|---|---|
| 동 | 비닐하우스 한 동 |
| 물리 배드 | 실제 농장에 존재하는 긴 재배 배드 |
| 논리 구역 | 시스템에서 난 묶음을 배치하고 관리하는 위치 단위 |
| 기본 논리 구역 | 물리 배드를 좌/우로 나눈 MVP 기본 구역 |

## 배드 정밀 배치 모델

논리 구역은 길이 방향의 `앞 구간`, `중간1~3`, `끝 구간`을 기본으로 가진다. 각 구간은 배치 규격과 화분 크기에 따라 여유·표준·확장·압축·임시 모드별 수용량과 허용 여부를 가진다.

난 묶음은 수량과 별도로 배치 규격, 판 수, 분할 배치 가능 여부를 가진다. 실제 배치 시 하나 이상의 구간에 수량과 판 수를 나누어 기록한다. 구간 정보가 없는 기존 난 묶음은 데이터 보존을 위해 미지정 상태로 유지한다.
| 행잉 구역 | 행잉 분화를 관리하기 위한 별도 구역 유형 |
| 난 묶음 | 같은 논리 구역 안에서 같은 품종/상태/규격으로 묶어 관리하는 난 그룹 |
| 작업 이력 | 농약, 비료, 분갈이, 잎 정리, 위치 이동 등 농장 작업 기록 |
| 작업 유형 | 작업 이력의 종류 |
| 위치 이동 | 난 묶음이 한 구역에서 다른 구역으로 이동한 작업 |
| 판매 전표 | 판매 내역을 기록하고 출력하는 문서 |
| 판매 품목 | 판매 전표에 포함되는 개별 품목 |
| 거래처 | 난을 구매하는 고객 또는 업체 |

## 14. 출하·경매 추적 도메인

기존 `SalesSlip`은 직접 판매 증빙으로 유지하고, 경매 출하는 별도 lot으로 추적한다.

```text
ImportBatch 1 ─ N ImportRow
AuctionShipment 1 ─ N AuctionShipmentLot
AuctionShipmentLot 1 ─ N AuctionAttempt 1 ─ N AuctionResultLine
AuctionShipmentLot 1 ─ N AuctionLotStatusHistory
```

- `AuctionShipmentLot`: 출하 수량, 판매 수량, 경매장 대기 수량, 반환 수량의 기준 단위
- `AuctionAttempt`: 유찰과 재경매를 포함한 경매 시도
- `AuctionResultLine`: 단가별 실제 경매 결과 원본 행
- `ImportRow`: CSV 원본과 정규화 결과, 자동 매칭 상태 보존
- `AuctionLotStatusHistory`: 반환 확인, 수량 보정, 수동 상태 변경 이력

자동 매칭은 `출하일 + 품종 + 등급 + 경매장`을 우선하고, 등급 누락 시 `출하일 + 품종 + 경매장`을 사용한다. 후보가 둘 이상이면 임의 선택하지 않고 확인 필요로 둔다.
