# 프로젝트 개요 v2.2

## 1. 프로젝트 목표

비닐하우스 난 농장을 관리하기 위한 웹 기반 농장 경영 시스템을 개발한다.

농장은 총 15개 동으로 구성되어 있고, 각 동에는 3개의 물리 배드가 존재한다. 실제 운영에서는 각 물리 배드가 중앙 관수 라인과 세로 철사 구조를 기준으로 좌/우 논리 구역처럼 활용된다. 따라서 1동은 운영상 6개의 기본 논리 구역을 가지는 것으로 볼 수 있다.

사용자는 위에서 농장을 내려다보는 농장맵 UI를 통해 동, 물리 배드, 논리 구역을 선택하고 해당 범위에 어떤 난 묶음이 있는지 확인할 수 있어야 한다. v2.2에서는 화면 역할을 명확히 나눈다. `농장 현황`은 전체 농장맵을 통한 통합 조회 화면으로 사용하고, `난 묶음 관리`는 선택한 동의 상세맵을 이용해 난 묶음을 생성·수정·이동하는 작업 화면으로 사용한다.

같은 동 안의 난 묶음 이동은 드래그 앤 드롭으로 처리한다. 다른 동으로 이동할 때는 드래그만으로 처리하지 않고, 목적 동/물리 배드/논리 구역을 단계적으로 선택하는 방식으로 처리한다. 농약·비료·분갈이·잎 정리·잡초 정리·단화 등의 작업 이력도 기록할 수 있어야 한다.

판매 내역은 기존에 전표로 관리되고 있으므로, MVP에서는 판매 전표를 디지털로 입력하고 A5 기준으로 출력할 수 있는 기능을 포함한다. 이후에는 판매 완료 또는 출고 완료 시 난 묶음 수량이 자동 차감될 수 있도록 확장한다.

---

## 2. 핵심 목표

이 프로젝트의 핵심 목표는 “완벽한 자동화”가 아니라 “현장에서 꾸준히 사용할 수 있는 쉬운 관리 시스템”이다.

중요한 판단 기준:

- 현장 작업자가 꾸준히 사용할 수 있어야 한다.
- 키보드 입력을 최소화하고 마우스/터치 기반 조작을 우선한다.
- 부모님 세대 사용자는 종이 출력물로 확인할 수 있어야 한다.
- 출력물은 A5를 기본으로 하며 글자가 작지 않아야 한다.
- 자식 세대 관리자는 PC/태블릿/모바일로 데이터를 입력하고 관리할 수 있어야 한다.
- 농장 운영 데이터가 장기적으로 쌓일 수 있어야 한다.
- 실제 농장의 물리 구조를 반영하되, MVP에서는 정밀 CAD 지도 수준의 복잡성을 피한다.
- 조회 화면과 작업 화면을 분리하여 실수로 데이터가 변경되지 않도록 한다.

---

## 3. 핵심 사용자

### 부모님 세대 작업자

- IT 기기 입력에는 익숙하지 않을 수 있음
- 키보드 타자보다 마우스 클릭, 터치, 큰 버튼 조작이 적합
- 종이 출력물, 전표, 장부 형태를 선호
- 복잡한 입력 화면은 사용하기 어려움
- 실제 농장 작업의 중심 사용자

### 자식 세대 관리자

- PC/태블릿/모바일을 이용해 데이터를 입력하고 관리
- 부모님이 작성한 수기 전표나 작업 내용을 시스템에 입력
- 농장 현황 화면을 통해 전체 상태를 조회
- 난 묶음 관리 화면을 통해 난 묶음 위치를 정리하고 이동 기록을 관리
- 필요 시 출력해서 부모님께 제공
- 시스템 유지보수 및 데이터 정리 담당

---

## 4. 운영 방식

초기 운영은 다음 흐름을 따른다.

```text
부모님이 기존 방식으로 수기 기록 또는 전표 작성
→ 자식 세대 관리자가 시스템에 입력
→ 필요한 출력물을 A5 기준으로 프린트
→ 부모님은 종이 출력물로 확인
→ 데이터는 시스템에 누적
```

농장 현황 조회는 다음 흐름을 따른다.

```text
농장 현황 화면에서 전체 농장맵 확인
→ 동 클릭 시 해당 동의 모든 난 묶음 목록 표시
→ 줌인 후 배드 클릭 시 해당 배드의 난 묶음 목록 표시
→ 더 줌인 후 논리 구역 클릭 시 해당 구역의 난 묶음 목록과 최근 작업 요약 표시
```

난 묶음 관리는 다음 흐름을 따른다.

```text
난 묶음 관리 화면에서 동 선택
→ 선택한 동 상세맵 표시
→ 논리 구역 선택
→ 난 묶음 추가, 수정, 삭제 또는 이동
→ 이동 시 위치 이동 이력 자동 기록
→ 구역별 관리표 출력
```

이 방식은 기존 업무 방식을 급격하게 바꾸지 않으면서 디지털 데이터를 축적하는 것을 목표로 한다.

## 5. MVP 범위

MVP에서 포함할 기능:

- 15개 동 관리
- 각 동의 3개 물리 배드 관리
- 각 물리 배드의 좌/우 기본 논리 구역 자동 생성
- 총 90개 기본 논리 구역 관리
  - 15동 × 3물리 배드 × 2논리 구역
- 농장 현황 전체맵 UI
- 전체 농장 보기
- 동/물리 배드/논리 구역 단계별 조회
- 선택 범위의 난 묶음 목록 표시
- 난 묶음 관리 상세맵 UI
- 선택한 동 상세 보기
- 동 상세 보기에서 물리 배드 좌→우 배치
- 논리 구역별 난 묶음 조회
- 난 묶음 등록/수정/삭제
- 난 묶음 구역 내 순서 관리
- 난 묶음 블록 UI 표시
- 같은 동 내부 난 묶음 드래그 이동
- 다른 동 이동 목적지 선택
- 난 묶음 이동 시 위치 이동 작업 이력 자동 생성
- 농약/비료/분갈이/상태 기록/메모 작업 이력 관리
- 잎 정리/잡초 정리/단화 작업 이력 관리
- 구역 또는 배드별 최근 작업일 표시
- 거래처 관리
- 판매 전표 관리
- 판매 품목 금액 자동 계산
- 판매 전표 A5 출력
- 구역/배드별 관리표 A5 출력

---

## 6. MVP 제외 범위

다음 기능은 MVP 이후로 미룬다.

- 사용자가 직접 논리 구역을 자유롭게 병합/분할하는 기능
- 자유 좌표 기반 영역 드래그 생성
- 철사 단위 정밀 위치 관리
- 15구/20구 판 단위 정밀 배치 관리
- 개별 화분 QR 코드 관리
- 일부 수량만 이동하여 난 묶음을 자동 분할하는 기능
- 여러 난 묶음 일괄 이동 또는 이동 바구니
- 판매 완료 시 난 묶음 수량 자동 차감
- 실시간 IoT 센서 연동
- AI 병해충 진단
- 모바일 앱 별도 개발
- 복잡한 권한 관리
- 외부 쇼핑몰 연동
- 회계 프로그램 연동
- 택배사 API 연동
- 정교한 GIS 지도 기능

단, 향후 확장이 가능하도록 데이터 구조와 컴포넌트 구조는 막아두지 않는다.
