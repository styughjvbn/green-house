# 배드 자리 차지 및 배치 추천 기능 정리

## 1. 기능 목적

배드 자리 차지 기능은 자리 정리 작업 시 난 묶음을 어느 배드와 구간에 배치할 수 있는지 판단하기 위한 기능이다.

이 기능의 목적은 정확한 자동 배치가 아니라, 실제 농장 오차와 현장 배치 방식을 고려해 배치 후보를 추천하고 작업자가 최종 확정할 수 있도록 돕는 것이다.

```text
정확히 몇 개가 들어가는가보다
어느 정도 무리하면 들어가는가를 보여주는 기능
```

---

## 2. 기본 판단

실제 농장에서는 같은 규격의 배드라도 다음 이유로 수용량이 달라진다.

```text
배드별 현장 오차
시작/끝 구간의 지지대 간섭
중간 통행 공간 일부 사용
15구/20구/24구 판 겹침 배치
화분 크기별 간섭
행잉 분화 또는 임시 배치
```

따라서 단순히 배드 길이와 판 크기로 자동 계산하지 않는다.

---

## 3. 배치 모드

배치 가능 여부는 하나의 수용량이 아니라 배치 모드별로 판단한다.

```text
여유 배치
- 통행과 작업 공간을 충분히 남기는 배치
- 장기 관리 권장

표준 배치
- 평상시 기본 배치
- 기본 추천 기준

확장 배치
- 통행 공간을 일부 줄이고 한 줄 더 배치
- 자리 정리 또는 단기 보관에 사용 가능

압축 배치
- 판을 일부 겹치거나 간격을 줄여 배치
- 단기 보관용
- 장기 관리 비추천

임시 배치
- 현장에서는 가능하지만 시스템상 경고가 필요한 배치
- 재정리 예정일과 메모 입력 권장
```

---

## 4. 배드 구간 개념

논리 구역 하나를 다시 길이 방향 구간으로 나눈다.

기본 구간은 다음처럼 구성할 수 있다.

```text
앞 구간
중간1
중간2
중간3
끝 구간
```

앞 구간과 끝 구간은 중간 구간보다 수용량이 적을 수 있다.
특히 지지대 간섭 때문에 특정 화분 크기나 특정 판 규격이 들어가지 않을 수 있다.

예시:

```text
1동 2배드 우측

앞 구간
- 20구 판 표준 2판
- 15구 판 표준 3판
- 5치 화분 불가
- 메모: 시작 지지대 간섭

중간1
- 20구 판 표준 5판
- 15구 판 표준 6판

끝 구간
- 20구 판 표준 2판
- 24구 판 압축 배치만 가능
- 5치 화분 불가
- 메모: 끝 지지대 간섭
```

---

## 5. 수용량 관리 방식

수용량은 다음 기준을 조합해 관리한다.

```text
논리 구역
구간
배치 형태
화분 크기
배치 모드
수용량
허용 여부
메모
```

예시:

```text
1동 2배드 우측 / 중간1 / 20구 판

여유 배치: 4판
표준 배치: 5판
확장 배치: 6판
압축 배치: 6판
```

```text
1동 2배드 우측 / 끝 구간 / 5치 단독 화분

허용 여부: 불가
메모: 지지대 간섭으로 배치 불가
```

---

## 6. 난 묶음 자리 차지 정보

난 묶음에는 실제 수량과 별도로 자리 차지 정보를 둔다.

```text
수량
배치 형태
판 수
화분 크기
분할 배치 가능 여부
```

예시:

```text
덴드로비움 C
수량: 192분
배치 형태: 24구 판
판 수: 8판
분할 배치: 가능
```

```text
고가 모주 묶음
수량: 20분
배치 형태: 단독 화분
화분 크기: 5치
분할 배치: 불가
```

## 7. 추천 결과 표시

추천 결과는 가능/불가만 표시하지 않고, 필요한 배치 강도를 함께 표시한다.

```text
추천
- 표준 배치 이내로 가능

가능
- 확장 배치까지는 가능

주의
- 압축 배치가 필요함
- 장기 관리 비추천

불가
- 압축 배치 기준도 초과
- 또는 해당 규격 배치 불가
```

예시:

```text
이동 예정: 덴드로비움 C / 24구 판 8개

추천
5동 1배드 우측
- 중간1 4판
- 중간2 4판
- 필요 배치: 표준

확장 필요
3동 2배드 좌측
- 앞 1판
- 중간1 4판
- 중간2 3판
- 필요 배치: 확장

압축 필요
7동 1배드 우측
- 중간2 5판
- 끝 3판
- 필요 배치: 압축
- 주의: 끝 구간 지지대 간섭

불가
1동 1배드 좌측
- 24구 판 배치 불가 구간 포함
- 압축 기준 초과
```

## 8. 작업 흐름

```text
1. 자리 정리 모드 진입
2. 이동할 난 묶음 선택
3. 배치 형태와 판 수 확인
4. 시스템이 구간별 남은 수용량 계산
5. 추천 후보 표시
6. 작업자가 현장에서 실제 배치
7. 실제 배치 위치와 배치 모드 확정
8. 위치 이동 이력 저장
9. 필요 시 재정리 예정일 저장
```

---

## 9. 현장 보정

추천 결과와 실제 현장이 다를 수 있으므로 작업 후 보정 기능을 제공한다.

```text
생각보다 좁음
계산과 비슷함
생각보다 여유 있음
```

이 피드백을 통해 구간별 수용량을 점진적으로 보정한다.

예시:

```text
추천: 표준 배치 가능
현장 결과: 안 들어감
보정: 표준 수용량 5판 → 4판
```

## 10. 데이터 구조 초안

### bed_zone_segments

논리 구역을 길이 방향 구간으로 나누는 테이블이다.

```text
id
bed_zone_id
name
segment_type
sort_order
memo
```

segment_type 예시:

```text
START
MIDDLE
END
CUSTOM
```

### bed_zone_segment_capacity

구간별 수용량과 배치 제약을 저장한다.

```text
id
bed_zone_segment_id
placement_type
pot_size
capacity_mode
capacity_value
is_allowed
memo
```

capacity_mode 예시:

```text
SPACIOUS
STANDARD
EXPANDED
COMPRESSED
TEMPORARY
```

### orchid_group_segment_placement

난 묶음이 실제로 어느 구간에 배치되었는지 저장한다.

```text
id
orchid_group_id
bed_zone_segment_id
quantity
tray_count
placement_mode
memo
```

이 테이블은 초기에는 생략할 수 있으며, 구간별 실제 배치 기록이 필요해지는 시점에 추가한다.

## 11. 단계별 구현 권장 순서

```text
1단계
논리 구역별 배치 모드 수용량 관리

2단계
앞/중간/끝 구간 관리 추가

3단계
구간별 화분 크기·판 규격 제약 추가

4단계
실제 구간 배치 기록 추가

5단계
자리 정리 추천 기능 구현

6단계
현장 피드백 기반 수용량 보정
```
