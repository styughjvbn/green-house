# 향후 확장 계획 v2

## 1. 논리 구역 병합/분할

MVP에서는 물리 배드별 좌/우 기본 논리 구역을 사용한다.

MVP 이후에는 현장 운영 방식에 맞게 사용자가 논리 구역을 직접 병합하거나 분할할 수 있도록 한다.

기능:

- 논리 구역 이름 변경
- 논리 구역 병합
- 논리 구역 분할
- 논리 구역 비활성화
- 사용자 정의 구역 생성

---

## 2. 세부 그리드 기반 배치

물리 배드의 실제 규격을 참고해 더 세부적인 관리용 그리드를 제공한다.

대상 정보:

- 물리 배드 길이
- 물리 배드 폭
- 철사 줄 수
- 받침대 간격
- 앞/중간/뒤 구간
- 좌/우 세부 구간

주의:

- 정밀 CAD 수준의 좌표 관리가 목적이 아니다.
- 현장에서 쉽게 이해할 수 있는 관리용 그리드를 우선한다.

---

## 3. 15구/20구 판 단위 관리

난 묶음을 판 단위로 관리할 필요가 생기면 배치 형태를 확장한다.

기능:

- 15구 판 등록
- 20구 판 등록
- 판 개수 기록
- 판별 수량 계산
- 판 단위 이동

---

## 4. 행잉 분화 관리

행잉 분화를 별도 논리 구역으로 관리한다.

기능:

- 행잉 구역 생성
- 행잉 난 묶음 등록
- 행잉 위치 메모
- 행잉 구역 작업 이력

---

## 5. 개별 화분 QR 코드 관리

고가품, 모주, 문제 개체를 중심으로 개별 화분 단위 관리를 추가한다.

기능:

- 개별 화분 등록
- QR 코드 생성
- QR 스캔 시 상세 페이지 이동
- 개별 화분 사진/작업 이력 관리
- 개별 화분 위치 이동 이력

---

## 6. 사진 기록

난 묶음 또는 개별 화분에 사진을 연결한다.

기능:

- 사진 업로드
- 날짜별 사진 기록
- 작업 이력과 사진 연결
- 개화 사진 기록
- 병해충 의심 사진 기록

---

## 7. 판매 전표와 수량 자동 연동

MVP에서는 전표 생성과 출력만 제공한다.

MVP 이후에는 판매 품목을 난 묶음과 연결하고, 판매 확정 또는 출고 완료 시 난 묶음 수량을 자동 차감한다.

기능:

- 판매 품목과 난 묶음 연결
- 출고 완료 시 수량 차감
- 전표 취소 시 수량 복구
- 일부 출고 처리
- 재고 변동 이력

주의:

- 전표 생성 시점에 바로 차감하지 않는다.
- 실제 출고 또는 판매 확정 상태를 기준으로 차감하는 것이 안전하다.

---

## 8. 사용자 정의 작업 유형

MVP 이후 현장 작업에 맞춰 작업 유형을 직접 추가할 수 있도록 한다.

기능:

- 작업 유형 생성
- 작업 유형 이름 수정
- 작업 유형 비활성화
- 작업 유형 정렬 순서 변경

---

## 9. IoT 센서 연동

비닐하우스 환경 데이터를 자동 수집한다.

대상 데이터:

- 온도
- 습도
- 광량
- 토양 또는 식재 수분
- CO₂

---

## 10. AI 병해충 진단

사진 기록이 충분히 쌓인 뒤 AI 진단을 검토한다.

주의:

- 초기부터 AI 진단을 목표로 하지 않는다.
- 먼저 사람이 판정한 사진과 상태 기록을 충분히 쌓는다.
- 데이터가 쌓인 후 병해충 의심 탐지 또는 상태 분류 기능을 검토한다.

---

## 11. 모바일 현장 입력

MVP는 PC/태블릿 중심으로 시작한다.

이후 현장 입력성이 중요해지면 모바일 UI를 강화한다.

기능:

- QR 스캔
- 빠른 작업 기록
- 사진 촬영 후 업로드
- 음성 메모

---

## 12. 통계/분석 기능

데이터가 쌓이면 다음 분석 기능을 추가한다.

- 품종별 판매량
- 거래처별 매출
- 월별 매출
- 논리 구역별 상태 이상 빈도
- 배드별 상태 이상 빈도
- 농약/비료 작업 주기
- 분갈이 후 생육 이상 발생률
- 위치 이동 후 상태 변화 추적
- 판매 가능 수량 예측

---

## 13. 외부 판매 채널 연동

추후 필요 시 외부 판매 채널과 연결한다.

후보:

- 스마트스토어
- 블로그
- 인스타그램
- 자체 쇼핑몰
- 거래처별 주문서

MVP에서는 제외한다.
